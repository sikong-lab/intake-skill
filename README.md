# 龙虾体检·需求问诊 Skill v3

> 把"说不清楚的需求"变成"结构化的诊断报告"

## 这是什么

一个AI问诊助手Skill，通过自然对话帮客户把模糊的需求变成结构化诊断报告。

**不是表单，是对话。** 它会追问、会挑战模糊表述、会帮你整理散装想法。

## v3 新特性

- **暂停/续接**：客户想不清楚可以随时暂停，下次说"继续问诊"自动恢复进度
- **自动保存报告**：对话结束后自动保存报告到文件，不用手动复制
- **弹性轮次**：不再锁死5-7轮，信息够了就收，不够就继续聊
- **分层知识库**：挑战规则、报告模板、D8详解拆到 references/ 目录，主文件更精简

## 目录结构

```
intake-skill/
├── SKILL.md                          # 主技能文件
├── README.md                         # 本文档
├── LICENSE                           # MIT 开源协议
├── .gitignore                        # 安全配置
└── references/
    ├── challenge-triggers.md         # 挑战追问规则表（11种触发条件）
    ├── report-template.md            # 诊断报告模板
    └── d8-best-practice.md           # D8最佳实践详解（关键维度）
```

## 核心能力

- **8维度信息采集**：从"你是谁"到"有没有最佳实践"，覆盖需求对齐所需的全部信息
- **挑战追问机制**：11种触发条件，遇到模糊表述自动追问细节
- **16分制评估**：实时追踪信息完整度
- **暂停/续接**：会话状态自动保存，支持跨会话继续
- **自动生成报告**：对话结束后自动保存Markdown报告文件

## 安装

### 方法一：让龙虾自己装（飞书妙搭用户）

在飞书对话里跟你的龙虾说：
> 帮我从 github.com/sikong-lab/intake-skill 安装这个skill。下载SKILL.md和references文件夹到skills/intake-skill/目录，注册到openclaw.json，然后重启gateway。

### 方法二：命令行安装

```bash
npx skills add sikong-lab/intake-skill
```

### 方法三：手动安装

1. 下载本仓库所有文件
2. 放到龙虾的 `skills/intake-skill/` 目录
3. 在 `openclaw.json` 的 `skills.entries` 中添加：
```json
"intake-skill": {
  "enabled": true,
  "path": "./skills/intake-skill/SKILL.md"
}
```
4. 重启 gateway

## 使用方法

安装后，说以下任意一句即可启动：
- "问诊"
- "需求诊断"
- "intake"
- "帮我搞清楚需求"

### 暂停和继续

- 随时说"暂停"/"先这样"/"下次再聊" → 进度自动保存
- 下次说"继续问诊" → 自动恢复上次进度

## 适用场景

- 咨询师接单前的需求对齐
- 服务商客户intake
- 任何"客户说不清需求 + 你怕理解偏差"的场景

## 设计理念

1. **问"现在怎么做的"，不问"你想要什么"** —— 断点比愿望有用
2. **D8最佳实践是关键维度** —— 决定是"补断点"还是"从零设计"
3. **不催客户** —— 想不清楚是正常的，给时间，支持暂停
4. **敢挑战模糊表述** —— 用触发规则表强制追问，不靠AI自己判断
5. **信息不够就标不够** —— 诚实 > 好看
