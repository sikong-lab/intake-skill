# 龙虾体检·需求问诊 Skill

> 把"说不清楚的需求"变成"结构化的诊断报告"

## 这是什么

一个AI问诊助手Skill，装到你的AI工具里后，它会通过5-7轮自然对话，帮你（或你的客户）把模糊的需求变成结构化的诊断报告。

**不是表单，是对话。** 它会追问、会挑战模糊表述、会帮你整理散装想法。

## 核心能力

- **8维度信息采集**：从"你是谁"到"有没有最佳实践"，覆盖需求对齐所需的全部信息
- **挑战追问机制**：遇到"差不多""比较急""帮我做就行"等模糊表述，自动追问细节
- **16分制评估**：实时追踪信息完整度，告诉你哪些信息够了、哪些还缺
- **自动生成报告**：对话结束后输出结构化Markdown报告，直接转发给服务方

## 适用场景

- 咨询师接单前的需求对齐
- 服务商客户intake
- 任何"客户说不清需求 + 你怕理解偏差"的场景

## 安装使用

### 方法一：通过 skills.sh 安装（推荐）

```bash
npx skills add sikong-lab/intake-skill
```

### 方法二：手动安装

1. 复制 `SKILL.md` 文件内容
2. 粘贴到你的AI工具的系统提示词/Skill配置中

## 使用方法

安装后，对AI说以下任意一句即可启动：

- "问诊"
- "需求诊断"
- "intake"
- "帮我搞清楚需求"

## 报告示例

问诊结束后会自动生成类似这样的报告：

```
# 龙虾体检·需求诊断报告

信息完整度：13/16
客户自评：已确认

## 客户画像
- 称呼：张老师
- 角色/行业：高职院校设计专业教师
- AI工具使用情况：用过DeepSeek做教案

## 断点
- 卡在哪一步：AI生成的教案格式和学校模板对不上
- 试过什么：手动复制粘贴到Word，但格式全乱

## 最佳实践判定
- 状态：有且知道（第1课已做好，格式标准都对）
- 方案方向：补断点（在已有模板上批量生成）

...
```

## 设计理念

这个Skill的核心设计原则：

1. **问"现在怎么做的"，不问"你想要什么"** —— 断点比愿望有用
2. **D8最佳实践是关键维度** —— 决定是"补断点"还是"从零设计"，搞反了会严重返工
3. **敢挑战模糊表述** —— 不靠AI自己判断，用触发规则表强制追问
4. **信息不够就标不够** —— 诚实 > 好看
