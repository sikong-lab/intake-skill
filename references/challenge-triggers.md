# 挑战触发规则表

遇到以下词汇/情况，**必须追问，不犹豫，不靠自己判断**：

| 触发条件 | 必须追问的方向 |
|---------|-------------|
| 客户说"格式有问题/格式不对" | 具体是哪种格式问题？行列？字体？结构？合并单元格？ |
| 客户说"用AI做过了/试过了" | 用哪个AI？做出来的东西还在吗？能发来看看吗？效果怎么样？ |
| 客户说"比较急/挺急的" | 具体截止日期是哪天？硬性的还是弹性的？过了会怎样？ |
| 客户说"有一些文件/有模板" | 文件在哪？什么格式？有没有一份你觉得"已经对了"的？ |
| 客户说"差不多/还行/基本可以" | "差不多"是哪里还差？"基本可以"是哪里不可以？ |
| 客户说"帮我做XX就行" | 先搞清楚现在卡在哪，不然做出来可能不是你要的 |
| 客户前后说法矛盾 | 直接指出矛盾，请对方澄清："你前面说XXX，但刚才又说YYY，能帮我确认一下吗？" |
| 客户提了3个以上方向 | 帮他列出来，请他选最急的一个先聚焦 |
| 客户说"我想要一个XXX" | 追问"你现在是怎么做的"——断点比愿望有用 |
| 客户说"都行/随便/你看着办" | 给2-3个选项让他选，帮他具体化 |
| 客户突然跳到新话题 | "这个我记下了。我们先把刚才那个聊完，免得两个都没聊透" |
| 客户说"找人做过/报过价/买过课" | "大概花了多少？效果怎么样？为什么没继续用？" |
| 客户说"给团队用/几个人一起用" | "大概几个人？每个人用的场景一样吗？" |
| 客户说"能不能做一个工具/系统/自动化" | "这个东西你多久用一次？是每天都用还是偶尔用？" |
