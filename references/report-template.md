# 需求诊断报告模板

对齐确认后，用以下模板生成报告并自动保存为文件。

```markdown
# 龙虾体检·需求诊断报告

生成时间：YYYY-MM-DD HH:MM
信息完整度：X/16
客户自评：已确认 / 部分确认 / 客户表示说不清楚

---

## 客户画像
- 称呼：
- 角色/行业：
- AI工具使用情况：

## 当前任务
- 在做什么：
- 用什么工具/流程：

## 断点
- 卡在哪一步：
- 试过什么，为什么没成功：

## 已有资产
- 手上有什么文件/半成品/模板：
- （如有）文件格式和内容概述：

## 期望产出
- 成品应该长什么样：
- 给谁看/交给谁：
- 有没有参照标准或模板：

## 最佳实践判定
- 状态：（有且知道 / 以为有但没有 / 不知道自己有 / 确实没有）
- 依据：（客户说了什么让你做出这个判断）
- 如果"有"：具体是什么？在哪里？（文件名、第几课、哪个版本）
- 方案方向：（补断点 / 从零设计 / 需要曹老师进一步判断）

## 约束条件
- 截止时间：
- 组织/格式/规格要求：

## 缺失信息（曹老师需要追问的）
- （列出0分和1分的维度，说明缺什么）

## 核心需求（一句话）
（用最直白的语言写，不要术语）

## 问诊结论

**提交建议：（三选一）**
- 建议提交（12分以上）
- 可以提交但有盲区（10-11分）
- 暂不建议提交（10分以下）

**以当前信息量，曹老师可以：**
- （列出能做什么）

**曹老师可能还会追问你：**
- （列出缺失维度的具体问题）

**你可以期待的下一步：**
- （给客户明确预期）

**未覆盖的需求（如有）：**
- （列出本次提到但未展开的需求方向，建议后续再约一次问诊）

---

## 📋 曹老师内参（客户不可见）

### 定价信号
- 替代选择：（客户试过什么？找过谁？花了多少钱？）
- 替代选择价格：（如果提到了具体数字）
- 任务频率：每天 / 每周 / 每月 / 每季度 / 一次性
- 单次时间成本：约X小时
- 紧迫度：高（有硬deadline）/ 中（想尽快）/ 低（没压力）
- 学习诉求：想学操作 / 想学方法论 / 想学思维方式 / 只要结果

### 复杂度预判
- 模块数估算：X个（列出来）
- 复杂度分区：🟢绿区(1-2模块) / 🟡黄区(3-5模块) / 🔴红区(6+模块或需API)
- 判断依据：（为什么这么判断）

### 能力边界
- 核心操作类型：管信息 / 处理媒体 / 混合
- 能力判定：🟢绿灯(能做) / 🟡黄灯(能做但要说清边界) / 🔴红灯(做不了)
- 如果黄/红灯：具体哪个环节超出能力？有没有补充路径？

### 客户分级
- 类型：个人轻需求 / 个人深需求 / 团队POC / 企业级
- 判断依据：（几个人用？什么场景？）

### 升单建议
- 建议报价区间：¥_____ - ¥_____
- 交付形态：Skill / HTML / 混合
- 报价话术锚点："这类服务市场上通常要[替代选择价格]元"
- 是否建议接单：是 / 否（原因）

### 不接判断（任中一条→建议不接）
- [ ] 替代选择价格 < 2000元（需求太轻）
- [ ] 算完报价 < 999元（低于底价）
- [ ] 需求本质是"帮我做一次"而不是"帮我建一个能复用的东西"
- [ ] 核心需求在红灯区且无补充路径

---
问诊轮次：X轮
是否有暂停/续接：是/否
```
