# 需求诊断报告模板

对齐确认后，用以下模板生成报告并自动保存为文件。

```markdown
# 龙虾体检·需求诊断报告

生成时间：YYYY-MM-DD HH:MM
信息完整度：X/16
客户自评：已确认 / 部分确认 / 客户表示说不清楚

---

## 客户画像
- 称呼：
- 角色/行业：
- AI工具使用情况：

## 当前任务
- 在做什么：
- 用什么工具/流程：

## 断点
- 卡在哪一步：
- 试过什么，为什么没成功：

## 已有资产
- 手上有什么文件/半成品/模板：
- （如有）文件格式和内容概述：

## 期望产出
- 成品应该长什么样：
- 给谁看/交给谁：
- 有没有参照标准或模板：

## 最佳实践判定
- 状态：（有且知道 / 以为有但没有 / 不知道自己有 / 确实没有）
- 依据：（客户说了什么让你做出这个判断）
- 如果"有"：具体是什么？在哪里？（文件名、第几课、哪个版本）
- 方案方向：（补断点 / 从零设计 / 需要曹老师进一步判断）

## 约束条件
- 截止时间：
- 组织/格式/规格要求：

## 缺失信息（曹老师需要追问的）
- （列出0分和1分的维度，说明缺什么）

## 核心需求（一句话）
（用最直白的语言写，不要术语）

## 问诊结论

**提交建议：（三选一）**
- 建议提交（12分以上）
- 可以提交但有盲区（10-11分）
- 暂不建议提交（10分以下）

**以当前信息量，曹老师可以：**
- （列出能做什么）

**曹老师可能还会追问你：**
- （列出缺失维度的具体问题）

**你可以期待的下一步：**
- （给客户明确预期）

**未覆盖的需求（如有）：**
- （列出本次提到但未展开的需求方向，建议后续再约一次问诊）

---
问诊轮次：X轮
是否有暂停/续接：是/否
```
