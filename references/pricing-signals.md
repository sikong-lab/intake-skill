# 定价信号采集指南

本文件供问诊助手在对话中自然采集定价相关信息，所有信息写入报告的"曹老师内参"板块，客户不可见。

---

## 3个必采信号

### 1. 替代选择（最重要的定价锚点）

**什么时候问**：聊D3（现有工作流）时自然带出

**话术参考（不要照读，融入聊天）**：
- "之前有没有试过别的方法？比如找人帮忙或者自己学？"
- "有没有了解过类似的服务或课程？大概什么价位？"
- "如果不用AI解决，你打算怎么办？"

**记录什么**：
- 客户提到的具体替代方案（课程/外包/助理/自己做）
- 如果提到价格，直接记录
- 如果没提价格，记录替代方案类型（后续曹老师可查市场行情）

### 2. 频率和量（判断值不值得定制）

**什么时候问**：聊D2（当前任务）时自然带出

**话术参考**：
- "这个事情你多久做一次？"
- "每次大概花多长时间？"
- "一次处理多少量？"

**记录什么**：
- 频率：每天/每周/每月/每季度/一次性
- 单次时间成本
- 处理量（如果适用）

### 3. 紧迫度（影响客户付费意愿）

**什么时候问**：聊D7（约束条件）时自然带出

**话术参考**：
- "这个事什么时候需要解决？"
- "有没有硬性的deadline？过了会怎样？"

**记录什么**：
- 高：有硬性deadline（具体日期）
- 中：想尽快但没硬性要求
- 低：没压力，什么时候都行

## 被动观察信号（不用主动问）

| 客户行为 | 记录为 |
|---------|--------|
| 提到"给团队用""几个人" | 客户类型→团队/POC |
| 提到"想学会""教我" | 学习诉求→想学操作/方法论 |
| 提到"帮我做好就行" | 学习诉求→只要结果 |
| 提到具体预算数字 | 直接记录 |
| 语气很急/反复催 | 紧迫度→高 |
| 需求涉及多个模块 | 复杂度→黄区或红区 |

---

## 复杂度分区速查

| 分区 | 信号 | 报价区间 | 交付形态 |
|------|------|---------|---------|
| 绿区 | 1-2个模块，单人用，不需要外部API | 3000元 | HTML或Skill |
| 黄区 | 3-5个模块，或需要简单API集成 | 5000元 | HTML+Skill混合 |
| 红区 | 6+模块，或需复杂API，或多人协作 | 8000-10000元 | 完整方案+API |

**快速判断法**：数客户提到的"动词"——每个不同的动作约等于一个模块。
- "帮我管客户+排日程" = 2个模块 = 绿区
- "管客户+排日程+翻译内容+生成报告" = 4个模块 = 黄区

## 能力边界速查

**一句话**：CC擅长"管信息的工具"，不擅长"处理媒体的工具"

| 判定 | 核心操作 |
|------|---------|
| 绿灯 | 信息管理、内容生成、数据处理、流程工具、文档工具、轻图片、API集成 |
| 黄灯 | 图表可视化、文件处理（Excel/PDF文字）、多人使用、手机端 |
| 红灯 | 视频剪辑、专业修图、3D、实时音视频、原生App、桌面软件 |

## 内部定价工作表模板

问诊结束后，在报告"曹老师内参"中填写：

```
替代选择：_______________（课程/外包/自己做/找助理）
替代选择价格：_______________元（如果知道）
任务频率：_______________
单次时间成本：_______________小时
紧迫度：高/中/低
学习诉求：操作/方法论/思维方式/只要结果

→ 建议报价 = min(替代选择价格 x 60-80%, 年度价值 x 20-30%)
→ 检查底线：≥ 999元
→ 交付形态：Skill / HTML / 混合
```
